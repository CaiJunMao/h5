数据转换

			隐式转换
				 * 在条件判断中null undefinded隐式转换为false，{}转换为true
				 * 数字和字符串的运算转换 
					 * 数字加字符串（空字符串也是“”） （运算符那里也有提到）
					 * 	如1+"1"结果为字符串“11”
					 * 	但-"1"结果为数字类型“-1”
				* 1、转换成布尔类型 undefined/null/NaN/""->false 不存在的东东转换成false
					undefined —> false
					null —> false
					数值       0,0.0,NaN —> false   其他数值—> true
					字符串    ""—>false    "etef" —> true
					其他对象—> true
					总结：存在的东东，转换成true， 不存在的东东转换成flase
				* 2、转换成数值类型 undefined —> NaN null/""->0 
						undefined —> NaN
						null —> 0
						字符串——>数值—>     "34234"—>34234,  "324sdfs3423" —> NaN,  "" —> 0
						布尔类型——>数值    true->1   false->0
						总结：能转成数字的就转成对应的数字。不能转成数字的要么0或1要么NaN
				* 3、转换成字符串类型
						undefined —>"undefined"
						null —>"null"
						布尔类型     true —>”true”     false —>"false"
						数值类型转成字符串    12321 —>"12321"   NaN —>"NaN"
						总结：字面值是什么就转成什么
			强制转换
			 	【Number（）】转为数字
					 * 1、null->0，但是用parseInt(null)却是NaN,null == undefined
					 * 2、undefined->NaN
					 * 3、bollean>数字
					 * 4、存在数字（也包括含小数点的数）以外的内容->NaN 
					 * 5、小数点、正负号、空格在开始位置不影响转化为数字 
					 * 但“2+3”->NaN
					 * 6、中间存在空格->NaN,全是空格=null->0					
					console.log(Number("a33"));//NaN
					console.log(Number(".33"));//0.33					
				【parseXxxx（）】 将内容转整数或浮点数
					 * 规则：
					 * 从第一个空白字符（除空格、tab、换行）开始转换，直到遇到非数字字符，转换失败则返回NaN		 
						console.log(parseInt("12.3"))//12
						console.log(parseInt("12.8"))//不会四舍五入
						console.log(parseInt("123a34"));//123
						console.log(parseFloat("12.3"));//12.3，浮点数
						console.log(parseFloat("12..3"));//12

					 * 参数2表示进制，如果不能识别则识别多少算多少
						console.log(parseInt("123",10));//转为10进制，123
						console.log(parseInt("123",2));
						//1，转为2进制，2和3超过二进制，所以只识别出1
						console.log(parseInt("12",3));//转为3进制，5
